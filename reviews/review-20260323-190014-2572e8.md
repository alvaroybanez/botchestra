## High

### Finding 1
- File: `.claude/review-loop-run-codex.sh`
- Line: `15`
- Severity: `high`
- Category: `Diff`
- Description: The review runner invokes `codex` with `--dangerously-bypass-approvals-and-sandbox`. That turns a repo-controlled review helper into an unrestricted execution path with full filesystem and network access driven by `.claude/review-loop-codex-prompt.txt`, which is a serious security footgun for local use and automation.
- Suggested fix: Remove the bypass flag from the default path. If unrestricted execution is ever needed, require an explicit opt-in environment variable or a separate manual script with a prominent warning.

### Finding 2
- File: `.claude/hooks/tdd_gate.py`
- Line: `97-129`
- Severity: `high`
- Category: `Holistic`
- Description: `track_test()` treats any successful `tsc`/`eslint`/`ruff`/`mypy` invocation as proof that every pending glue file is verified, then clears `pending_glue` wholesale. A scoped command such as `eslint src/foo.ts` would incorrectly mark unrelated staged glue files as verified and let them pass the commit gate.
- Suggested fix: Classify lint/typecheck commands by scope the same way test commands are classified. Only clear all pending glue files for full-suite runs, map targeted runs back to affected files, and add regression coverage for targeted and unknown-scope commands.

## Medium

### Finding 4
- File: `.claude/review-loop-codex-prompt.txt`
- Line: `10`
- Severity: `medium`
- Category: `Diff`
- Description: The diff-review instructions claim to cover all uncommitted changes, but they only prescribe `git diff` and `git diff --cached`, which do not include untracked files. In the current tree, the new review-loop files and PRD files are untracked, so the review can silently miss a large part of the pending work.
- Suggested fix: Extend the prompt to enumerate untracked files explicitly, for example with `git status --short` or `git ls-files --others --exclude-standard`, and require those files to be reviewed alongside the normal diffs.

### Finding 5
- File: `.claude/review-loop-codex-prompt.txt`
- Line: `10`
- Severity: `medium`
- Category: `Diff`
- Description: The prompt hardcodes `git diff HEAD~5`, but this repository currently has only one commit (`69f06b4` on March 23, 2026), so that revision does not exist and the command fails with an ambiguous revision error. The review loop is therefore brittle on fresh or shallow histories.
- Suggested fix: Guard the history lookup with `git rev-parse --verify HEAD~5` and fall back to the root commit or `git diff HEAD` when fewer than five commits exist.
